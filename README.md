# سایت ساخت توکن روی Polygon

سایتی که کاربران با پرداخت کمیسیون (به MATIC یا USDT) می‌توانند توکن ERC-20
دلخواه خودشان را روی شبکه Polygon بسازند و مستقیماً به کیف پول خودشان دریافت
کنند. این نسخه پس از یک بازبینی امنیتی کامل اصلاح شده است.

## ساختار پروژه

```
contracts/
  ERC20Template.sol          ← قالب توکن (کلون می‌شود، خودش مستقیم دیپلوی نمی‌شود)
  TokenFactory.sol           ← قرارداد اصلی: کمیسیون می‌گیرد و توکن می‌سازد
  mocks/
    MockERC20.sol            ← توکن جعلی برای تست (شبیه‌ساز USDT)
    MaliciousFeeReceiver.sol ← قرارداد برای تست حمله reentrancy
test/
  TokenFactory.test.js       ← تست‌های کامل امنیتی و عملکردی (Hardhat + Chai)
scripts/
  deploy.js                  ← اسکریپت دیپلوی روی تست‌نت/مین‌نت Polygon
backend/
  server.js                  ← سرویس گوش‌دادن به رویدادها + API تاریخچه توکن‌ها
frontend/
  CreateToken.jsx            ← فرم React برای کاربر
```

## بازبینی امنیتی انجام‌شده (نسخه دوم)

| مشکل | ریسک | راه‌حل اعمال‌شده |
|---|---|---|
| امکان `initialize()` مستقیم روی implementation | بالا | `_disableInitializers()` در constructor قالب |
| بدون محدودیت طول name/symbol | متوسط | حداکثر ۵۰ کاراکتر برای نام، ۱۱ برای نماد |
| تغییر کمیسیون بین ارسال و تایید تراکنش کاربر | متوسط | پارامتر `maxFee` + بازگرداندن خودکار مابقی وجه |
| نبود توقف اضطراری | متوسط | افزوده شدن `Pausable` با `pause()`/`unpause()` |
| ریسک گم‌شدن مالکیت با انتقال یک‌مرحله‌ای | متوسط | تغییر به `Ownable2Step` |
| آدرس صفر در constructor | پایین | `require` روی همه پارامترهای آدرس |
| گیر کردن دارایی ارسالی اشتباه | پایین | تابع `rescueERC20`/`rescueNative` + رد ارسال مستقیم MATIC |
| فرض اشتباه ۱۸ رقم اعشار برای USDT | بالا (کاربردی) | تابع `usdtDecimals()` + مستندسازی (USDT روی Polygon معمولاً ۶ رقمی است) |
| Reentrancy از طریق `feeReceiver` مخرب | بالا | `nonReentrant` روی هر دو تابع ساخت توکن (تست اختصاصی دارد) |

تست‌ها همه این موارد را پوشش می‌دهند (`test/TokenFactory.test.js`).


اگر خطایی گرفتید (مثلاً به‌خاطر تفاوت نسخه کتابخانه‌ها)، برایم بفرستید تا اصلاح کنم.

## مراحل راه‌اندازی کامل

1. **نصب:**
   ```bash
   npm install
   cp .env.example .env
   # سپس مقادیر .env را با اطلاعات واقعی خودتان پر کنید
   ```

2. **تست روی تست‌نت Polygon Amoy (حتماً قبل از mainnet):**
   ```bash
   npm run deploy:amoy
   ```
   برای تست‌نت باید یک `MockERC20` با ۶ رقم اعشار به‌جای USDT واقعی دیپلوی
   کنید و آدرسش را در `.env` به‌عنوان `USDT_ADDRESS` بگذارید.

3. **دیپلوی روی mainnet:**
   ```bash
   npm run deploy:mainnet
   ```
   آدرس‌های چاپ‌شده (Template و Factory) را نگه دارید.

4. **راه‌اندازی بک‌اند ثبت تاریخچه:**
   ```bash
   # FACTORY_ADDRESS و FACTORY_DEPLOY_BLOCK را در .env با خروجی مرحله ۳ پر کنید
   npm run backend
   ```
   API در دسترس خواهد بود:
   - `GET /api/tokens` — تاریخچه کامل
   - `GET /api/tokens/by-creator/:address` — توکن‌های ساخته‌شده توسط یک کیف پول
   - `GET /api/tokens/:tokenAddress` — جزئیات یک توکن

5. **اتصال فرانت‌اند:** آدرس Factory دیپلوی‌شده را در `frontend/CreateToken.jsx`
   جایگزین `0xYourFactoryAddressHere` کنید.

## نکات امنیتی که همچنان باید خودتان رعایت کنید

- **آدیت حرفه‌ای:** قبل از mainnet، حتماً یک آدیت مستقل (حتی پایه) توسط متخصص امنیت قرارداد هوشمند انجام دهید. کد بالا ریسک‌های شناخته‌شده را پوشش می‌دهد اما جایگزین آدیت حرفه‌ای نیست.
- **کلید owner:** حتماً از کیف پول چندامضایی (Gnosis Safe) برای مالکیت قرارداد فکتوری استفاده کنید، نه یک کیف پول تک‌کلیدی.
- **مانیتورینگ:** رویداد `TokenCreated` و تغییرات `FeesUpdated`/`FeeReceiverUpdated` را زیر نظر بگیرید (مثلاً هشدار Telegram/Discord هنگام تغییر feeReceiver).
- **Rate limiting در بک‌اند:** برای جلوگیری از سوءاستفاده از API عمومی، محدودیت نرخ درخواست (مثلاً با `express-rate-limit`) اضافه کنید.
